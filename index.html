Permissions Feature

2026

The brief was to design a permissions feature that enabled parents and carers to control their child's access to certain features within the app. The challenge was to create a secure approval process that felt straightforward and efficient, allowing permissions to be granted or revoked without introducing unnecessary friction for either user.

As the lead designer, I was responsible for designing the end-to-end user experience, ensuring the journey was intuitive for both the child requesting permission and the parent authorising it. The final solution guided the child through the request process, while parents could quickly approve access using either a QR code or a four-digit code. Permissions could be revoked at any time, giving parents ongoing control while keeping the overall experience simple, fast, and easy to understand.

Push Notification UI Design

2026

The brief was to redesign the push notifications interface to make it easier for users to manage, review, and interact with notifications. The existing experience relied on a grid of notification cards, making it difficult to scan, locate, and compare notifications efficiently. The redesign focused on improving usability by introducing a searchable, sortable vertical list alongside a contextual preview of the destination page.

As the lead designer, I was responsible for the end-to-end UX and UI design process. I explored user needs and pain points, restructured the information architecture, and designed a more intuitive layout that prioritised efficiency and clarity. The final solution replaced the card-based grid with a streamlined notification list, while a live preview panel provided immediate context by displaying the page each notification linked to, reducing the need for users to navigate away from the interface.
